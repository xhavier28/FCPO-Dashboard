--- DESIGN DECISIONS ---

1. TM threshold = 0.50
   At threshold=0.50, TM regime-risk exit rarely fires (0-3 exits
   per window). Selected as best worst-case config.

2. Cost model
   Cost is deducted ONCE per full position close, per lot.
   cost_points = cost_myr / POINT_VALUE
   For scale-in: total_cost = total_lots * cost_per_lot

3. MtM Sharpe (adj Sharpe)
   For OLD cost validation: uses 100 MYR flat for all instruments
   (matching original code exactly for reproducibility).
   For NEW cost: uses differentiated costs per instrument type.

4. Panel caching
   Data panel (4070 rows, 2008-2026) built once, cached in-memory.
   PM/TM predictions (~2016 each) also cached.

5. Dashboard pre-caching
   Locked-config daily results saved to locked_config_cache.pkl.
   Intraday results computed on demand (fast: no PM/TM recomputation
   needed, uses cached panel).

6. Scale-in UI
   3 independent tier rows, each with its own enable checkbox,
   z-level input, and lot-size input. Enabling tier 1 and 3 but
   not 2 is supported (per-tier independence).

7. Two-tab layout
   Tab 1: "Daily (4 Walk-Forward Windows)" — all 4 windows side
          by side, worst-case Sharpe highlighted.
   Tab 2: "60-Min Intraday (2024-2026)" — single continuous range.
   Parameters are SHARED across both tabs.

8. Intraday exit priority
   Daily-level exits (shape change, time stop, TM regime-risk) are
   checked FIRST at the start of each day using daily settlement.
   If no daily exit, intraday bars are scanned for TP, SL, and
   scale-in triggers within that day.
