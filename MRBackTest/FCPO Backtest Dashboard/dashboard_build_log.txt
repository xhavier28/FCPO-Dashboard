--- DESIGN DECISIONS ---

1. TM threshold = 0.50
   At threshold=0.50, TM regime-risk exit rarely fires (0-3 exits
   per window). This was selected as the best worst-case config.

2. Cost model
   Cost is deducted ONCE per full position close, per lot.
   cost_points = cost_myr / POINT_VALUE
   For scale-in: total_cost = total_lots * cost_per_lot

3. MtM Sharpe (adj Sharpe)
   For OLD cost validation: uses 100 MYR flat for all instruments
   (matching original code exactly for reproducibility).
   For NEW cost: uses differentiated costs per instrument type.

4. Panel caching
   The data panel (4070 rows, 2008-2026) is built once and cached
   in-memory. PM/TM predictions (~2016 each) are also cached.

5. Dashboard pre-caching
   Locked-config results are saved to locked_config_cache.pkl
   to avoid ~2-minute PM/TM computation on every page load.

6. Scale-in implementation
   - 0-3 independently toggleable tiers
   - Scale-in checks fire BEFORE exit checks each day
   - Direction-aware: SHORT entries scale at z >= tier_level,
     LONG entries scale at z <= -tier_level
   - All lots exit together; cost = total_lots * cost_per_lot

7. Loss bucket classification
   A = invalidated (regime break)
   B/C = time_stop (B=adverse extension, C=stalled — combined
         because max|z| tracking not yet in engine)
   D = regime_risk
   TP-loss = take_profit with net_pnl <= 0 (cost-induced)
   SL = stop_loss
